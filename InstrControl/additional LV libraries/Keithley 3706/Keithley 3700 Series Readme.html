Keithley 3700 Series Instrument Driver Readme

1. Overview

Instrument Driver Technology: LabVIEW Plug and Play
Manufacturer: Keithley Instruments
Supported Language(s): LabVIEW 8.0 or greater
Supported Model(s): Keithley 3706, 3706A
Model(s) Tested: Keithley 3706, 3706A
Interface(s): GPIB, USB, Ethernet
Firmware Revision Tested: 1.5.0

Certified: No
Keithley Supported: Yes
Source Code Available: Yes

Driver Revision: 1.1.2
Original Release Date: 9/14/2007
Current Revision Date: 10/07/2011

2. Required Software

Some software components need to be installed before using this instrument driver. The minimum versions of these components are listed below, and can be downloaded from the Download Site.

VISA 4.0 or later


For more details, refer to the LabVIEW Help under Help » VI, Function, & How-To Help.

3. Known Issues

To report issues or provide feedback about this instrument driver, please send an email to applications@keithley.com.

4. Revision History

The latest version of this and other LabVIEW instrument drivers can be downloaded at the Keithley Instruments Website.

REV 1.1.2, 10/07/2011
Keithley Instruments, Inc.

  • Fix instrument pallet issue.

 

REV 1.1.1, 6/07/2011
Keithley Instruments, Inc.

  • Updated supported models to include the 3706A.

 

REV 1.1.0, 7/09/2010
Keithley Instruments, Inc.

  • Changed the driver from the LLB format to a LabVIEW Plug and Play format.
  • Added support for 3750 Multifunction I/O Card.
  • Added support for TSP Script handling with the TSP Script Loader, Run TSP Script, Delete TSP Script, Terminate TSp Script and Get TSP Script Catalog VIs.
  • Added support for 3732 Quad 4x28 High Density Reed Matrix Card.
  • Added support for 3724,3731,3732 and 3750 Pseudocards.
  • Corrected error in the Configure Scan Vi  Scan Count parameter. Added Measure Count input.
  • Added support for Appending Multi Channel Scan Steps in the Scan List VI using the scan.addimagestep() command.
  • Added support for simplified channel creation with the Channel Create Specifier.vi using the channel.createspecifier() command.
  • Corrected error in Configure Measurement Vi with setting Auto Zero control to ‘On’.
  • Updated documentation errors.
